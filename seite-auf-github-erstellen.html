Seite auf GitHub erstellen

Matthias Ulpmer, Ivan Bacher

1) Token erstellen

Um eine Seite auf GitHub zu erstellen, muss man zunächst ein sog. “Zugangs-Token” erstellen. Damit kann Publii auf die Github-Seite zugreifen und Änderungen vornehmen, so dass es die Website erstellen und bearbeiten kann, sobald man in der Publii-App auf den Sync-Button klickt.

Anleitung:

  1. Zunächst meldet man sich bei GitHub an und klickt auf die Profil-Einstellungen.
  2. Anschließend scrollt man ganz hinunter und klickt auf Developer settings
  3. Danach klickt man auf  “Personal Access tokens” → “Tokens (classic)”
  4. Dann klickt man auf “Generate new token” und danach auf “Generate new token (classic)”
  5. Als Nächstes gibt man einen Namen, wie z.B. "Publii GitHub Page" ein. Im Abschnitt "Select scopes"  muss man ein Häkchen bei der Option "repo" setzen. Zudem kann man auch ein Ablaufdatum für das Token festlegen, z. B. "90 Tage". (Den Rest belässt man beim Standard). Anschließend generiert man das Token.

 

Generiertes Token:

Wichtig: Das Token kopieren/speichern, denn danach kann man es nicht mehr ansehen.

2) Repo für die Webseite erstellen

  1. Man klickt auf seine Repositories → Neues Repository erstellen. Anschließend gibt man einen Namen ein, lässt das Repo auf öffentlich und erstellt dieses. (Man wählt keine zusätzliche Option aus)
  2. Anschließend auf den Link README klicken und diese (ohne Änderungen) committen.
  3. Anschließend wechselt man auf den Reiter Einstellungen → Pages und setzt dort den Branch auf “main”.
  4. Wenn man nun die Seite neu lädt und die Seite angezeigt wird, ist die Einrichtung abgeschlossen.
