Das Repo mit Publii verknüpfen

Matthias Ulpmer, Ivan Bacher

Folgende Informationen müssen angegeben werden:

  • Website Url - Dies ist der Pfad zum Github-Repository
  • API-Server - Standardmäßig ist dieses Feld bereits mit dem Standard-API-Server ausgefüllt, dies kann man so belassen
  • Username / Organization - Github-Benutzernamen eingeben.
  • Repository - Geben Sie hier den Namen Ihres Github-Repositories ein (in unserem Fall gh-pages).
  • Branch - In unserem Fall ist der namen des Branches “main”.
  • Token - Den zuvor erstellten Token einfügen
  • Parallele Uploads - Der Standardwert ist '1'. Legt die Anzahl der Upload-Streams fest, die Publii beim Hochladen der Inhalte der Website auf GitHub Pages verwenden kann. Höhere Zahlen ermöglichen schnellere Uploads, können aber bei langsamen Verbindungen zu Fehlern führen oder einen Fehler 403 aufgrund von API-Upload-Ratenbeschränkungen verursachen.
  • API-Ratenbegrenzung - Diese Option nur deaktivieren, wenn man Github Enterprise mit deaktivierter API-Ratenbegrenzung verwendet. Andernfalls kann die Deaktivierung dieser Option zur Bereitstellung Fehlern führen.

Sobald alle Optionen eingestellt sind,die Änderungen speichern und anschließend auf Verbindung testen drücken, um zu prüfen, ob Publii eine Verbindung zu Ihrer Github-Seite herstellen kann.

Anschließend klickt man “Sync your website”, um die Webseite hochzuladen.

WICHTIG! Die hochgeladene Publii-Website ist möglicherweise nicht sofort sichtbar, wenn der Synchronisierungsprozess abgeschlossen ist. Es kann ca. 1 Minute dauern, bis die Änderungen auf Ihrer Website sichtbar sind.
